## 🔧 工作原理

### 流程1: 第一次进入页面

```
onLoad()
  ├─ loadCachedSeats()      ← 立即从本地显示 (<100ms) ✨
  ├─ loadSeats()            ← 后台获取最新 (2-3秒后完成)
  ├─ startSeatPolling()     ← 启动20秒轮询
  └─ loadAnnouncementsAsync() ← 异步加载公告 (500ms后)
```

**用户感受**: 
- 立即看到座位列表 (来自缓存)
- 无需等待网络请求
- 2-3秒后数据自动更新

### 流程2: 返回此页面

```
onShow()
  ├─ getUserInfo()          ← 更新用户信息
  ├─ 检查needRefresh标志
  │  ├─ 有标志 → loadSeats()  (从座位页预约回来)
  │  └─ 无标志 → 显示缓存      (其他返回)
  └─ isFirstLoad标记 → 跳过重复加载
```

**对比**:
- ❌ 优化前: 每次都loadSeats() → 3-4秒等待
- ✅ 优化后: 智能判断 → 需要才刷新 → 无等待

## 💻 代码示例

### 缓存优先策略

```javascript
// 优化前 - 只能等待
onLoad() {
  this.loadSeats();  // 3-4秒
}

// 优化后 - 即时显示
onLoad() {
  this.loadCachedSeats();  // <100ms - 从文件系统读取
  this.loadSeats();        // 2-3秒 - 后台更新
}
```

### 异步加载

```javascript
// 优化前 - 串行执行（总时间=相加）
onLoad() {
  this.loadSeats();          // 需要 2秒
  this.loadAnnouncements();  // 需要 1秒
  // 总时间: 3秒
}

// 优化后 - 并行执行（总时间=最长的）
onLoad() {
  this.loadSeats();                // 2秒 (主流程)
  this.loadAnnouncementsAsync();   // 1秒 (后台)
  // 总时间: 2秒 (快33%)
}
```

### 智能刷新

```javascript
// 优化前 - 重复加载
onLoadX() { loadSeats(); }
onShow()  { loadSeats(); }  // 重复了！

// 优化后 - 避免重复
onLoad() {
  loadCachedSeats();  // 立即显示
  loadSeats();        // 后台更新
  isFirstLoad = true;
}

onShow() {
  if (needRefresh) {
    loadSeats();      // 必要时才加载
  } else if (isFirstLoad) {
    // 跳过，使用轮询或下拉刷新
  }
}
```

## 🔍 故障排查

### Q: 为什么没有缓存数据？
**A**: 第一次使用时数据会在首次API调用后缓存，第二次进入就有了。

### Q: 缓存数据过旧怎么办？
**A**: 系统会在后台自动更新，用户无需操作。同时每20秒还会轮询刷新。

### Q: 返回座位页面时没有看到更新？
**A**: 这是正常的！除非：
- 你预约了座位（会自动刷新）
- 20秒轮询时间到
- 你主动下拉刷新

### Q: 怎样强制刷新？
**A**: 在座位页面**下拉即可**，会立即刷新最新数据。
