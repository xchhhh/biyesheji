## 🔌 API接口集成

### 1. 座位查询接口
```javascript
// 请求
GET /api/reservations/seats/1?date=2024-03-20

// 响应 (200)
{
  "total": 100,
  "available": 45,
  "occupied": 50,
  "maintenance": 5,
  "seats": [
    { "id": 101, "seat_number": "A1", "status": 0 },
    { "id": 102, "seat_number": "A2", "status": 1 },
    ...
  ]
}

// 状态: 0=可选, 1=已预约, 2=维护, 3=我的
```

### 2. 预约提交接口
```javascript
// 请求
POST /api/reservations/reserve
{
  "seat_id": 101,
  "room_id": 1,
  "reservation_date": "2024-03-20",
  "time_slot": "08:00-10:00"
}

// 成功响应 (200)
{
  "reservation_id": 1001,
  "seat_number": "A1",
  "status": 0
}

// 冲突响应 (409)
{
  "code": 409,
  "message": "座位已被他人预约"
}

// Token失效 (401)
{
  "code": 401,
  "message": "登录已过期，请重新登录"
}
```

### 3. 错误处理流程
```
发送请求
  ↓
收到响应
  ├─ 200: 成功，处理数据
  ├─ 400: 参数错误，显示提示
  ├─ 401: Token失效，跳转登录
  ├─ 409: 座位冲突，刷新列表
  ├─ 500: 服务器错误，显示重试
  └─ -1: 网络错误，自动重试
```

## 📞 常见问题解答 (FAQ)

### Q1: 小程序无法连接后端怎么办？
**A:** 
1. 确保后端服务运行 (`python run.py`)
2. 检查 `config.js` 中 API_BASE_URL 是否正确
3. 在 `project.config.json` 中确保 `"urlCheck": false` (开发环境)
4. 检查防火墙是否阻止 5000 端口

### Q2: 座位网格显示太密集，看不清楚怎么办？
**A:** 修改 `seats.wxss` 中的尺寸：
```css
.seat {
  width: 80rpx;   /* 从60增大 */
  height: 80rpx;
}

.seats-grid {
  grid-gap: 12rpx;  /* 从8增大 */
}
```

### Q3: 如何处理座位预约冲突？
**A:** 当收到409错误时，小程序会：
1. 显示"座位已被他人预约"
2. 自动刷新座位列表
3. 让用户选择其他座位

### Q4: Token过期时怎样处理？
**A:** 当收到401错误时：
1. 清除本地存储的token
2. 自动跳转到登录页面
3. 用户需要重新登录

### Q5: 性能不足怎么优化？
**A:**
- 启用座位列表缓存 (5分钟TTL)
- 虚拟滚动 (仅显示可见座位)
- 分页加载 (100个座位分页)
