# 座位预约状态实时更新配置

## 📊 实时更新机制

管理后台已配置了**自动实时更新功能**，可以让值班人员实时查看座位占用情况的变化。

---

## ✅ 已实现的实时更新

### 1️⃣ 值班快查面板 (最常用)
- **更新频率**: 每 5 秒自动刷新一次
- **何时启动**: 用户点击"值班快查"选项卡时启动
- **何时停止**: 用户切换到其他选项卡时停止
- **显示内容**:
  - 各房间座位占用情况
  - 占用率百分比
  - 空闲座位数
  - 维修座位数
  - 最后更新时间（带绿色实时指示灯）

**使用方式**:
```
1. 登录管理后台
2. 点击左侧菜单 "值班快查"
3. 看到页面上方显示 "数据最后更新于: HH:MM:SS ●"
4. 数据会每5秒自动更新一次
```

### 2️⃣ 控制面板 (Dashboard)
- **更新频率**: 每 10 秒自动刷新一次
- **何时启动**: 用户在控制面板时组动启动
- **何时停止**: 用户切换页面时停止
- **显示内容**:
  - 系统用户总数、活跃数、禁用数
  - 今日预约、签到、完成数
  - 座位总数、占用、维修数
  - 待处理维护数
  - 最近操作日志

**使用方式**:
```
1. 登录管理后台后自动进入控制面板
2. 观察卡片中的数据
3. 每10秒数据会自动更新
4. 无需手动刷新页面
```

如果要修改更新频率，可以编辑 `app/static/js/admin.js` 文件：

```javascript
// 在 admin.js 第 11 行找到：

## 📡 更新机制详解

### 值班快查实时更新流程
```
用户进入值班快查
    ↓
立即加载一次数据 (loadDutyDashboard)
    ↓
启动定时器 (setInterval)
    ↓
每5秒自动执行 loadDutyDashboard
    ↓
获取最新的座位占用数据
    ↓
更新页面显示 + 显示更新时间
    ↓
用户切换其他页面时停止定时器
```

### 数据流程
```
浏览器 ─(AJAX请求)→ /api/admin/duty-dashboard
       ←(JSON响应)─ Flask后端
       
获取数据包括:
- 各房间座位总数
- 当前占用座位数
- 当前空闲座位数
- 正在维修的座位数
- 占用率计算
- 服务器时间戳
```

页面顶部会显示:
```

                          ↑

**绿点说明**:

### 颜色编码
在房间详细视图中：
- 🟢 **绿色**: 空闲座位
- 🟡 **黄色**: 占用座位
- 🔴 **红色**: 维修座位

## 🧪 测试实时更新

### 如何验证实时更新是否工作

1. **打开管理后台值班快查**
   ```
   http://localhost:5000/admin/
   点击 "值班快查"
   ```

2. **观察更新时间**
   - 看顶部是否显示 "数据最后更新于: HH:MM:SS"
   - 打开浏览器开发者工具 (F12)
   - 打开 Console 标签
   - 应该看到提示: "✓ 已启动值班面板实时更新（每5秒刷新一次）"

3. **观察数据变化**
   - 记下当前的占用座位数
   - 等待5秒
   - 观察数字是否变化（如有新预约或签到）
   - 更新时间应该改变

4. **测试定时器启停**
   - 切换到其他选项卡（如用户管理）
   - Console 中应不再输出更新日志
   - 重新点击值班快查
   - 定时器重新启动

## ⚡ 性能优化建议

### 高性能部署建议

1. **减少更新频率** (服务器压力大时)
   ```javascript
   const DUTY_DASHBOARD_REFRESH_INTERVAL = 10000; // 改为10秒
   ```

2. **使用WebSocket** (对于大规模部署)
   - 当前使用 AJAX 轮询
   - 可升级为 WebSocket 长连接
   - 减少HTTP请求开销

3. **启用浏览器缓存**
   - 确保在 Flask 配置中启用缓存头
   - 减少重复传输相同数据

### 网络使用估计
- **当前轮询**: 每5秒1个请求 = 720个请求/小时
- **每个请求**: 约1-2 KB 数据
- **带宽占用**: 约7-14 MB/小时
