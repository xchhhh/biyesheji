### 方案 B：专业 Locust 测试（用于详细分析）

#### 方式 1：命令行模式（推荐）

```bash
# 运行标准配置：100 个用户，每秒增加 10 个，运行 60 秒
locust -f test_locust_load.py --host=http://localhost:5000 -u 100 -r 10 --run-time 60s --headless
```

- `-u 100` : 最终并发用户数
- `-r 10` : 每秒增加的用户数（缓坡）

## ❌ 常见问题

### Q1：连接超时错误
**错误**：`Connection refused: [连接被拒绝]`

**解决方案**：
1. 确认后端服务正在运行
   ```bash
   python run.py  # 在另一个终端中
   ```
2. 检查服务是否在 `http://localhost:5000` 运行
3. 修改脚本中的 API_BASE_URL

### Q2：数据库连接错误
**错误**：`Database connection error`

**解决方案**：
1. 检查数据库是否启动
2. 检查 `.env` 文件中的数据库配置
3. 根据需要执行数据库初始化

### Q3：成功率过低（< 95%）
**现象**：大量失败请求

**解决方案**：
1. 检查测试数据是否正确生成
2. 验证座位是否充足
3. 检查认证 Token 有效性
4. 减少并发用户数重新测试

### Q4：响应时间过长（P99 > 2000ms）
**现象**：后端响应缓慢

**解决方案**：
1. 检查数据库性能
2. 检查 Redis 缓存是否启用
3. 增加应用实例或使用负载均衡
4. 优化数据库查询

### Q5：内存占用过高
**现象**：Python 进程占用大量内存

**解决方案**：
1. 减少并发用户数
2. 减少 RESERVATIONS_PER_USER
3. 检查是否有内存泄漏
