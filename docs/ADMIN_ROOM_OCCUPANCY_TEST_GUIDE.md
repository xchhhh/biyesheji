## 步骤1：后端 API 验证

### 1.1 测试 API 端点

打开命令行或使用 Postman，执行以下请求：

```bash
# Windows PowerShell
curl -H "X-Admin-Token: admin_test_token" http://localhost:5000/api/rooms/occupancy

# Linux/Mac
curl -H "X-Admin-Token: admin_test_token" http://localhost:5000/api/rooms/occupancy
```

**预期响应：**
```json
{
  "code": 0,
  "data": [
    {
      "room_id": 1,
      "room_name": "309自习室",
      "floor": 3,
      "occupancy_rate": 0.65,
      "occupied_seats": 26,
      "total_seats": 40,
      "available_seats": 14
    },
    ...
  ]
}
```

✅ **验证项：**
- [ ] 响应状态码为 200
- [ ] code 为 0
- [ ] data 数组不为空
- [ ] 每个房间都有必需的字段

## 常见问题排查

### Q1: 看不到房间卡片
**可能原因**：
1. API 返回 code 不是 0
2. 数据库中没有房间数据
3. 容器 ID 不匹配

**解决方案**：
1. 在控制台查看 Network 请求，检查 API 返回的响应
2. 检查数据库中 `reading_room` 表是否有数据
3. 检查 HTML 中 `id="rooms-occupancy-container"` 是否存在

### Q2: 卡片显示但不更新
**可能原因**：
1. 定时器未启动
2. 离开仪表板时定时器被清理
3. 浏览器缓存问题

**解决方案**：
1. 查看控制台是否有启动消息
2. 不要切换标签页，停留在仪表板上观察6秒
3. 按 Ctrl+Shift+Delete 清除浏览器缓存，重新加载

### Q3: 占用率计算错误
**可能原因**：
1. 后端计算错误
2. 前端乘以 100 后四舍五入有误

**解决方案**：
1. 手动计算验证：occupied_seats / total_seats
2. 检查 JavaScript 代码中的 `toFixed(0)` 是否正确

### Q4: 样式显示不正常
**可能原因**：
1. CSS 文件未加载
2. Bootstrap 类名错误
3. 浏览器兼容性问题

**解决方案**：
1. 在浏览器 DevTools 的 Sources 中确认 CSS 文件是否加载
2. 检查 HTML 中的 class 属性拼写
3. 尝试用其他浏览器测试
