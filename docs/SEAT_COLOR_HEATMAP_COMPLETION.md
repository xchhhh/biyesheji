## 📞 常见问题解答

**Q1: 为什么预约后颜色改变这么快？**
A: 因为我们改为『本地立即更新』而不是『等待 API 刷新』。预约成功后，立即更新前端的座位队象，用户看不到任何延迟。

**Q2: 热力表的数据是从哪来的？**
A: 当前使用模拟数据。数据来自 processSeatData() 中的 `seatData.heat` 或随机值。未来可以接入真实的预约数据。

**Q3: 其他用户能看到我预约的座位颜色改变吗？**
A: 其他用户下次打开座位选择页面时会看到最新数据。我们没有实现实时推送（可以后期优化）。

**Q4: 热力表会影响性能吗？**
A: 完全不会。热力表只是显示，数据计算很轻量级，不会影响页面性能。

**Q5: 如何关闭热力表？**
A: 在 seats.js 的 data 中设置 `showHeatmap: false` 即可隐藏。
