# 🔧 座位维修申请 - 快速启动

## 🎯 5分钟快速开始

### 1️⃣ 重启前端应用

后端服务已在运行。请在WeChat开发者工具中：
- 关闭当前预览
- 刷新/重新编译项目
- 确保能看到最新的代码

### 2️⃣ 登录账户

- 打开mini-program
- 登录或注册账户
- 保持登录状态

### 3️⃣ 访问维修申请页面

**方法一：从"我的"页面进入**
```
底部导航栏 → "我的" → 向下滚动 → "我的信息" → "🔧 座位维修申请"
```

**方法二：直接导航**
```javascript
wx.navigateTo({
  url: '/pages/maintenance/maintenance'
});
```

### 4️⃣ 提交维修申报

1. 切换到"申报维修"标签页
2. 输入座位号：
   - 手动输入：A-001, A001, 或 1100
   - 或点击📱按钮扫描二维码
3. 选择问题类型（5选1）
4. 选择严重程度（4选1）
5. 填写问题描述（至少5个字）
6. 可选：输入联系电话
7. 点击"提交维修申报"

### 5️⃣ 查看申报进度

1. 切换到"申报历史"标签页
2. 查看所有维修申报记录
3. 按状态筛选（全部/待处理/处理中/已完成）
4. 点击卡片查看详细信息

## 🎨 功能介绍

### 申报维修流程

| 步骤 | 说明 |
|------|------|
| 1️⃣ **座位号输入** | 输入座位编号，支持扫一扫 |
| 2️⃣ **问题分类** | 选择问题类型（5种） |
| 3️⃣ **优先级设置** | 设置严重程度（4个等级） |
| 4️⃣ **详细描述** | 尽可能详细地描述问题 |
| 5️⃣ **联系方式** | 留下电话以便管理员联系 |
| 6️⃣ **快速提交** | 一键提交给管理员 |

### 问题类型

- 🔨 **座位损坏** - 座位、扶手、靠背等损坏
- 🧹 **座位脏污** - 座位不洁净、有污渍
- 🪑 **家具问题** - 其他家具相关问题
- ⚡ **电气问题** - 照明、插座等电气问题
- ❓ **其他问题** - 其他未分类的问题

### 严重程度

| 等级 | 颜色 | 说明 | 处理优先级 |
|------|------|------|----------|
| 低 | 🟢 绿色 | 轻微影响 | 最低 |
| 中 | 🟠 橙色 | 中等影响 | 中等 |
| 高 | 🔴 红色 | 严重影响 | 高 |
| 严重 | 🟣 紫色 | 无法使用 | 最高 |

### 维修状态

| 状态 | 说明 | 图标 |
|------|------|------|
| ⏳ 待处理 | 已提交，等待管理员处理 | 橙色 |
| 🔧 处理中 | 管理员正在处理维修 | 蓝色 |
| ✅ 已完成 | 维修已完成 | 绿色 |
| ❌ 已取消 | 申报已被取消 | 灰色 |

## 🔍 常见问题

### Q: 座位号怎么填？
**A:** 支持以下格式：
- `A-001` (标准格式)
- `A001` (无分隔符)
- `1100` (座位ID)

系统会自动识别并转换。

### Q: 问题描述写多少字最好？
**A:** 
- 最少：5个字（太短无法说明问题）
- 最佳：50-200个字（清楚说明问题）
- 最多：500个字（不要超过）

详细的描述有助于管理员快速解决问题。

### Q: 联系电话必须填吗？
**A:** 不是必填，但**强烈建议填写**。
- 方便管理员主动联系
- 确认维修位置
- 通知维修完成

### Q: 如何查看维修进度？
**A:** 
1. 进入"申报历史"标签页
2. 最新的申报显示在最上面
3. 下拉查看历史申报
4. 点击卡片查看详细信息

### Q: 能修改已提交的申报吗？
**A:** 不能直接修改，但可以：
- 新提交一份修正的申报
- 与管理员电话联系更新信息

### Q: 维修需要多久？
**A:** 取决于问题的严重程度：
- **低** - 1-2个工作日
- **中** - 几个小时到1天
- **高** - 优先处理
- **严重** - 立即处理

## 🚀 最佳实践

### 1. 详细的描述很重要
```
❌ 不好: "座位坏了"
✅ 好: "座位右边扶手折断了一半，坐下时会不稳定"
```

### 2. 选择准确的严重程度
```
低 → 轻微问题，不影响使用
中 → 轻度影响，影响舒适度
高 → 严重影响，有安全隐患
严重 → 无法使用，立即需要修复
```

### 3. 留下电话号码
```
管理员可以：
- 确认问题位置
- 通知维修完成时间
- 询问额外信息
```

### 4. 定期检查进度
```
在"申报历史"标签页中：
- 看到"处理中" → 维修人员在忙
- 看到"已完成" → 可以继续使用
- 超过1周还是"待处理" → 可以直接联系管理员
```
